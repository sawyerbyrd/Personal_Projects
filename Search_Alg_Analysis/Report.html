# creates a 'number' of random matricies (as dataframes) of size 'size'
def rand_matrix(size, number):
    matricies = []
    for _ in range(0, number):
        matrix = np.random.randint(1, 1000, size=(size, size))
        symmetric_matrix = np.triu(matrix) + np.triu(matrix, 1).T
        np.fill_diagonal(symmetric_matrix, 0)
        df = pd.DataFrame(symmetric_matrix)
        matricies.append(df)
    return matricies

The Algorithm that provides the best cost seems to mostly be Repeated Randomized Nearest Neighbor with 2-Opt. From the graph we can see that NN and NN2O are pretty much the same when it comes to cost and number of nodes expanded vs size of graph. NN2O is slightly better due to the fact that it takes the NN solution and optimizes it. One notable point is how as the size of the graphs increase, the total cost seems to decrease for NN and NN2O, aproaching the cost solution for RNN. This could be because as the number of nodes increases, the "middle" of the optimal path doesnt matter as much.

Another observation can be made about the number of nodes expanded. All three algorithms seem to follow the same linear increase in the number of nodes expanded. In fact, by checking the data that I printed out, the number of nodes is exactly the same across the three. They also increase by the same amount each time, 5x+4

When it comes to runtimes, the time complexity of NN2O increases much faster (exponentially) with the size of the graph than the other two algorithms. This is due to the fact that it has to loop through each row and column -> N^2, on top of running NN first.

RNN calls NN2O after each iteration so you would immagine that it's runtime and nodes expanded would be greater than that of NN2O. However since the amount of restarts is 0, it only does this once and there are unlikely to be many optimizations to be made.
